Fin mai 2026, Anthropic a publié un article intitulé « When AI builds itself ». Le titre est discret, mais le contenu est le signal le plus important de l'industrie du codage IA en 2026 : Claude écrit désormais plus de 80% du code mergé en interne chez Anthropic, la productivité trimestrielle des ingénieurs a été multipliée par 8, Claude Mythos Preview atteint une accélération de 52x sur les tâches de code d'entraînement, et le taux de succès sur les tâches ouvertes grimpe à 76%.

## Quatre chiffres : Anthropic utilise l'IA pour écrire l'IA

L'équipe d'Anthropic a publié une divulgation interne dévastatrice. Le chiffre de 80% est particulièrement frappant : cela signifie que la majorité du code qui entre dans la base de code d'Anthropic — l'organisation qui construit Claude — est désormais écrite par Claude lui-même. Ce n'est plus un assistant, c'est l'auteur principal.

## Pourquoi l'amélioration récursive n'a abouti qu'en 2026

L'amélioration récursive auto (RSP) théorisée dès les années 1950 nécessitait trois conditions préalables que seules les architectures 2025-2026 ont satisfaites : (1) Contexte long suffisant (1M+ tokens) pour comprendre des bases de code entières ; (2) Capacité de métacognition (Claude peut évaluer son propre code) ; (3) Infrastructure d'entraînement en boucle fermée (RLHF + RLAIF).

## Impact structurel pour les développeurs

La transition s'opère en trois phases : Complétion (2022-2024, l'IA suggère la ligne suivante), Agent Loop (2025, l'IA itère en boucle write→test→fix), Auto-amélioration récursive (2026+, l'IA écrit le code qui entraîne la prochaine génération d'IA).
