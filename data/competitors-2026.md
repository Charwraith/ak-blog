## 竞品 1：MiniMax 直连（platform.minimax.io）

### 基本信息

| 维度 | 详情 |
|------|------|
| 产品形态 | 模型厂商直连 API |
| 模型数 | 8 个文本模型 + 视频/语音/图像/音乐 API |
| 定价模式 | 按 token 计费（pay-as-you-go） |
| 用户量级 | 面向全球开发者（中文+英文文档） |
| 核心定位 | 高性能+低成本的国产模型 API |

### 定价分析

| 模型 | 输入 ($/1M tokens) | 输出 ($/1M tokens) | 上下文 | 缓存读取 |
|------|-------------------|-------------------|--------|---------|
| M2.5 | $0.15 | $0.90 | 197K | $0.03 |
| M2.7 | $0.25 | $1.00 | 205K | $0.05 |
| M3 | $0.30 | $1.20 | 1M | $0.06 |
| M2.1 | $0.29 | $0.95 | 197K | $0.03 |

- **最贵输出**：M1 ($2.20) / M2 Her ($1.20)
- 1M 以上长上下文（M3/M1/MiniMax-01）有价格翻倍机制（输入 512K 后从 $0.30→$0.60）
- 缓存读取价 $0.03-0.06/M tokens（约标准价的 20%）

### 限流策略

| 模型 | RPM | TPM |
|------|-----|-----|
| M3 | 200 | 10M |
| M2.7/M2.5/M2.1/M2 | 500 | 20M |

- 视频生成仅 5 RPM，非常严格
- 文本模型限流属于中等水平（对比 Anthropic 的 spend-based tier 较宽松）

### 稳定性

- 中国公司运维，国内访问极稳定
- 海外通过自有 CDN 覆盖
- 截至 2026-06-17，8 个模型全部活跃在线

### 中国可访问性

- **国内：畅通无阻**（本土公司，无需翻墙）
- **海外：部分地区有延迟**，但整体可用
- 文档中英双语，开发者社区偏中文

### Affiliate / 推荐政策

- **无公开 Affiliate Program**
- 无推荐返佣机制
- 无渠道合作伙伴分层
- 无 302.ai 等三方聚合者的官方合作

| 定价 | ★★★★☆ | 极其便宜（M2.5 $0.15输入），但长上下文翻倍机制需注意 |

## 竞品 2：OpenRouter（openrouter.ai）

### 基本信息

| 维度 | 详情 |
|------|------|
| 产品形态 | LLM API 聚合网关 |
| 模型数 | 400+（来自 70+ 提供商） |
| 定价模式 | 原生定价 + 5.5% 平台费 |
| 用户量级 | 全球开发者社区，活跃度中等 |
| 核心定位 | 多模型统一接入+自动路由+失败回退 |

### 定价分析

| 维度 | 详情 |
|------|------|
| 免费版 | 25+ 免费模型，50 请求/天，20 RPM |
| 付费版 (PAYG) | 5.5% 平台费，≥$10 信用后无限速 |
| BYOK | 每月 100 万免费请求，之后 5% 费用 |
| 企业版 | 协商折扣，SLA，SSO |
| 最低充值 | $0.80 |

**定价模式关键词**：`No markup on provider pricing` + `5.5% flat fee` + `Zero Completion Insurance`

### 限流策略

| 层级 | 限流 |
|------|------|
| 免费用户 | 50 请求/天，20 RPM |
| 付费 ($10+ 信用) | 付费模型无限速，免费模型 1000 请求/月 |
| BYOK | 100 万请求/月免费，之后正常 |

- 自带自动回退（fallback）机制——首个提供商故障时自动切换替代模型
- Zero Completion Insurance：只对成功的请求计费

### 稳定性

- **约 25-40ms 边缘延迟**
- 多提供商故障切换（provider 等级故障时自动 fallback）
- SOC 2 Type I 认证（2026 年中）
- 状态页：status.openrouter.ai

### 中国可访问性

- **海外畅通，国内不稳定**——服务器不在中国
- 国内用户需翻墙或通过本地网关中转
- 支持多家中国提供商（DeepSeek、Qwen、GLM 等），但模型调用仍走海外路由
- 文档纯英文，支付需要信用卡/加密币

### Affiliate / 推荐政策

- **有 Affiliate Program**（推荐购买信用可获得返佣——但具体比例未公开）
- 社区口碑传播是主要获客渠道（Reddit/HN 推荐较多）
- 无公开的渠道/分层合作伙伴计划

### 综合评估

| 维度 | 评分 | 说明 |
|------|------|------|
| 定价 | ★★★★☆ | 5.5% 平台费较透明，Zero Completion 有趣，但 BYOK 5% 隐藏成本 |
| 模型数 | ★★★★★ | 400+ 模型，70+ 提供商，行业最多 |
| 限流 | ★★★★☆ | 付费后基本无限速，免费版限制严格 |
| 稳定性 | ★★★★☆ | 多提供商故障切换基本可靠 |
| 中国可访问性 | ★★★☆☆ | 国内需翻墙，海外体验好 |
| Affiliate | ★★★☆☆ | 有但未大推 |

## 竞品 3：Portkey（portkey.ai）

### 基本信息

| 维度 | 详情 |
|------|------|
| 产品形态 | AI 网关 + 可观测性平台 |
| 模型数 | 1600+（通过网关间接支持） |
| 定价模式 | 按日志记录量计费（SaaS）+ 开源版免费 |
| 用户量级 | 企业级客户（Qoala/Ario/Haptik/Fortune 500 Pharma） |
| 核心定位 | 生产级 AI 控制面板（观测/治理/优化） |

### 定价分析

| 方案 | 价格 | 请求量 | 特点 |
|------|------|--------|------|
| 开源版 | 免费 | 无限（自托管） | 基础网关功能 |
| 开发者版 | 免费 | 10K 日志/月 | 3 天日志保留 |
| 生产版 | $49/月 | 100K 日志/月 (+$9/100K) | 30 天保留 + 语义缓存 |
| 企业版 | 自定义 | 10M+ 日志/月 | VPC/SOC2/GDPR/HIPAA |

**关键注意**：Portkey 不直接提供模型推理——它是网关层，按**日志记录量**计费，不是按 token。

### 限流策略

| 层级 | 日志限制 |
|------|---------|
| 开发者版 | 10K 条/月（超过只影响日志记录，不影响请求） |
| 生产版 | 100K 条/月 + $9/100K overage（最多 3M） |
| 企业版 | 自定义 |

### 稳定性

- 作为网关层，稳定性取决于底层模型 + Portkey 代理层
- 开源版自托管：取决于自我运维
- SaaS 版：Uptime SLA（Enterprise 协议内约定）
- Portkey 位于模型 API 和用户代码之间——增加了一层延迟
- 缓存机制（语义缓存+简单缓存）可大幅降低延迟和成本

### 中国可访问性

- **海外 SaaS 服务，国内不可直接访问**
- 企业版支持 VPC / Private Cloud Deployment
- 文档纯英文
- 开源版可在中国自托管（GitHub 仓库可用）

### Affiliate / 推荐政策

- **无公开 Affiliate Program**
- 主要靠企业销售团队直接获客
- 有 Partner/Integration 生态（与 GitHub Actions / LangChain / LlamaIndex 等集成）

### 综合评估

| 维度 | 评分 | 说明 |
|------|------|------|
| 定价 | ★★★☆☆ | $49/月起步不贵但上限低（3M overage 封顶），企业版不透明 |
| 模型数 | ★★★★★ | 网关模式间接支持所有模型 |
| 限流 | ★★★★☆ | 按日志量而非请求量，设计灵活 |
| 稳定性 | ★★★☆☆ | 网关层增加延迟，需依赖底层模型稳定性 |
| 中国可访问性 | ★★☆☆☆ | SaaS 不可用，仅开源版自托管 |
| Affiliate | ★☆☆☆☆ | 无 |

## 对 akrmio 项目的启示

### 1. 差异化机会

| 维度 | 市场空白 | akrmio 定位 |
|------|---------|-------------|
| 中国+海外双端 | MiniMax 只有中国，OpenRouter/Portkey 只有海外 | **双端覆盖** |
| 定价透明 | OpenRouter 5.5% 隐藏、Portkey 企业不透明 | **全透明定价** |
| 中国模型聚合 | 无专门聚合中国模型的网关 | **中国模型第一选择** |
| 支付兼容 | 三者均不支持多种跨境支付 | **Stripe+加密+支付宝/微信** |
| Fallback 延迟 | OpenRouter fallback 后切换速度无 SLA | **毫秒级路由+预热** |

### 2. 定价策略建议

- **对标 MiniMax** 做中国模型聚合：比 MiniMax 直连略贵（+5-10%）但提供多模型+fallback
- **对标 OpenRouter** 做海外模型：比 OpenRouter 的平台费低（3% vs 5.5%）
- **对标 Portkey** 做网关功能：开源策略吸引开发者，SaaS 版按量收费
- **核心杠杆**：双端可访问性（国内不翻墙+海外低延迟）是独一无二的优势

### 3. 风险提示

- OpenRouter 已显示 Model Ownership（Krutrim）模式——akrmio 若深度依赖上游模型，需考虑上游封禁风险
- Portkey 开源版功能有限——网关功能的自托管控件需要完善
- MiniMax 可考虑直接合作：成为 MiniMax 在海外/三方聚合的官方渠道
